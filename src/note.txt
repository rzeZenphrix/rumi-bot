# Continuation Prompt: Discord Bot Feature Audit, Fixes, and Command Expansion

Continue from the current codebase and the work already completed. Do **not** restart the project, rewrite working systems unnecessarily, or duplicate commands that already exist.

Before implementing anything new, audit the existing implementation and classify each requested feature as:

- `Already implemented`
- `Partially implemented`
- `Broken`
- `Missing`
- `Blocked`

Skip features that are already fully implemented. Fix features that are broken or incomplete. Implement missing features in practical batches.

Every response should include meaningful implementation progress. Aim to complete **20–30 features or commands per message** whenever possible. If a feature is large, split it into smaller working parts and complete as many as possible.

## Immediate Priorities

### 1. First-Person Bot Replies

All user-facing bot replies should be written in first person wherever natural.

Examples:

- `I updated this server’s configuration.`
- `I added that warning.`
- `I couldn’t find anything for that.`
- `I saved your embed template.`
- `I don’t have permission to do that yet.`

Apply this to:

- Command responses
- Error messages
- Success messages
- Setup flows
- Dashboard messages
- Moderation logs where appropriate
- Embed previews
- Validation errors
- AI-generated bot replies

### 2. Persistent Supabase Storage

The bot must use persistent Supabase storage instead of memory-only state.

Store long-term data in Supabase, including:

- Guild configuration
- Command settings
- Fake permissions
- Moderation cases
- Warnings
- Automod rules
- Anti-raid settings
- Anti-nuke settings
- Ticket data
- Leveling data
- Economy data
- Embed templates
- Custom commands
- Welcome messages
- Leave messages
- Boost messages
- Logs configuration
- User preferences
- Premium status
- Scheduled tasks
- Reminders
- Media references
- Dashboard auth/session data where needed

Requirements:

- Add idempotent migrations.
- Add typed database helpers or repository functions.
- Supabase should be the source of truth.
- Caches are allowed, but they must not replace persistent storage.
- Data must survive bot restarts.
- Handle Supabase errors gracefully.
- Add diagnostics for DB connectivity.
- Avoid duplicate rows with proper unique constraints.
- Validate imported/exported config data before saving.

### 5. Fix Captioning Links, GIFs, and Tenor GIFs

The media caption system must correctly handle links, GIFs, and Tenor URLs.

Problem case:

```txt
https://tenor.com/view/caught-reference-bro-niche-eagle-gif-14350895293905603987
```

Current bug:

- The bot returns it as a GIF attachment.
- The final result is still static.

Required behavior:

- Detect Tenor links.
- Resolve Tenor share links into direct animated media.
- Prefer animated GIF, MP4, WebM, or animated WebP where supported.
- Do not caption a static thumbnail if animated media is available.
- Preserve animation after captioning.
- Support Discord CDN images.
- Support direct image URLs.
- Support GIF URLs.
- Support Tenor URLs.
- Support common media hosts.
- If animation cannot be preserved, explain the limitation in first person.
- Add tests or validation cases for animated GIF handling.

## Embed Script Compatibility

The bot must support Bleed/Greed-style embed scripting.

Support this style:

```txt
{embed}
$v{description: ...}
$v{color: #000000}
$v{image: https://example.com/image.png}
$v{author: {user.name} && {user.avatar}}
$v{field: name && value && inline}
$v{message: content}
```

Also support:

- `,ce` shortcut
- Multiple embeds in one script
- Message content plus embeds
- Variables
- Invisible characters
- Emojis
- Animated emojis
- Markdown formatting
- Buttons
- Dropdowns
- Template saving/loading
- Script previews
- Script validation
- Script formatting

Variables should include at least:

- `{user.mention}`
- `{user.name}`
- `{user.avatar}`
- `{user.id}`
- `{server.name}`
- `{server.id}`
- `{channel.name}`
- `{channel.id}`

The parser must preserve spacing, formatting, emojis, invisible characters, animated emojis, and markdown.
