# Non-Negotiable Core Rules

## Audit and Safety Rules

1. Inspect files before editing them.
2. Do not replace the entire system unless a file is clearly broken beyond repair.
3. Do not duplicate commands, aliases, event handlers, services, repositories, migrations, or dashboard routes.
4. Preserve working code.
5. Fix broken systems in place when possible.
6. If a feature is already implemented and working correctly, leave it alone.
7. If a feature is partially implemented, complete it using the existing architecture.
8. If a feature is broken, repair it without creating parallel systems.
9. If a feature is missing, implement it using reusable helpers.
10. If something is blocked by missing secrets, missing APIs, missing permissions, or missing external services, clearly mark it as `Blocked`.

## Database and Supabase Rules

1. Supabase is the persistent source of truth.
2. Do not hardcode SQL inside command files, route files, event handlers, or feature logic.
3. Move SQL into dedicated `.sql` schema/migration files.
4. Migrations must be idempotent.
5. Use typed repository/helper functions for all database reads and writes.
6. Validate all inputs before saving to Supabase.
7. Add graceful handling for Supabase outages, DNS failures, invalid credentials, and network errors.
8. No database failure should crash the bot.
9. Features that require the database should fail gracefully with a clear user-facing message.
10. Add useful logs for failed database operations without leaking secrets.

## Discord Safety Rules

1. Validate Discord permissions before actions.
2. Validate role hierarchy before moderation actions.
3. Add fake-permission support for moderation/admin commands.
4. Add cooldowns and rate limits to abuse-prone commands.
5. Add pagination for long outputs.
6. Add clear user-facing failure messages.
7. Do not allow the bot to silently fail.
8. Cleanly handle deleted roles, deleted channels, missing permissions, invalid settings, and unavailable guild resources.

Example user-facing failure style:

```txt
I couldn’t lock this channel because I’m missing Manage Channels.
```

```txt
I couldn’t jail this member because their highest role is above mine.
```

```txt
I couldn’t save this setting because the database is currently unreachable.
```

## User-Facing Reply Style

1. Keep bot replies clean and professional.
2. Use first person where natural.
3. Avoid raw stack traces in Discord replies.
4. Use embeds only where they improve readability.
5. Use normal messages where explicitly requested.
6. Improve UI for economy, fun, social, text, bookmark, calendar, color, crypto, domain, and utility commands.
7. Use canvas/image generation where appropriate for visually rich commands.
